# C#.NStar
**Язык программирования C#.NStar** - ультра-высокоуровневый, кортежно-ориентированный, транспайлируемый, статически типизированный <small><small>(за некоторыми исключениями)</small></small>, Си-подобный язык программирования. Он использует <a href="https://github.com/Etyuhibosecyu/.NStar">фреймворк .NStar</a>, который можно использовать и без C#.NStar, а наоборот мало смысла. Впрочем, о фреймворке написано по ссылке в предыдущем предложении, здесь же описание самого языка.<br>

## Основные отличия C#.NStar от C#.NET
**1.** Можно объявлять классы внутри функций. Более того, вся программа представляет собой одну большую функцию, поэтому в C#.NStar нет понятия "функция Main", такой функцией является вся программа.<br>
**2.** Нет массивов, они были упразднены как слишком низкоуровневые. Вместо этого используются списки - <tt>() T list</tt>. Также есть оператор для создания многомерных списков - <tt>(n) T listOfLists</tt>.<br>
**3.** Другие правила определения строк. В "обычных" строках двойная кавычка экранируется комбинацией <tt>\\q</tt>, сам символ <tt>\\</tt> - <tt>\\!</tt>. Также есть "сырые" строки, начинающиеся символами <tt>/"</tt> и заканчивающиеся символами <tt>"\\</tt>, внутри которых можно писать эти же символы при условии соблюдения парности, а также verbatim-строки и просто кавычки, вообще без экранирования.<br>
**4.** Допускающие вложенность комментарии, начинающиеся комбинацией /{ и заканчивающиеся на }/.<br>
**5.** Не нужно городить простыню кода для выполнения блока кода, записанного в строку, это делается элементарно: <tt>ExecuteString(string, args)</tt>.<br>
**6.** Нет ввода-вывода в консоль, он был упразднен как устаревший. Есть однократный **вывод** (без ввода) в поток, в котором запущен транслятор (а этим потоком может быть и консоль), с помощью ключевого слова return, и есть полноценный ввод-вывод в GUI (мы используем современный фреймворк AvaloniaUI).<br>
**7.** Другие формы циклов: доступен цикл <tt>loop</tt>, выполняющийся потенциально бесконечно, пока не будет явно прерван оператором <tt>break</tt> (вместо <tt>while (true)</tt>), цикл <tt>loop { ... } while (condition);</tt>, отличающийся от простого <tt>while</tt> тем, что в том возможно такое условие, что цикл не выполнится ни разу, а этот хотя бы раз, но выполнится, упрощенный цикл <tt>for</tt> - вместо кучи сложных конструкций, в которых можно запутаться, если вы новичок и не знакомы с древними языками - <tt>for (i in a..b)</tt>, а также <tt>repeat (n)</tt>, который, надеюсь, не нужно пояснять, и их комбинации <tt>for (...) while (condition)</tt> и <tt>repeat (n) while (condition)</tt>, повторяющиеся, пока или один из составляющих комбинацию циклов завершится, или другой (ну или оба одновременно).<br>

## Основные возможности, отсутствующие в C#.NStar по сравнению с C#.NET (для исправления в будущем)
**1.** Сложный, многопрофильный полиморфизм.<br>
**2.** Экстенты.<br>
**3.** Сокращенная форма объявления метода - <tt>type Function(parameters) => expression;</tt>.<br>
**4.** Структуры, интерфейсы и другие "типы типов".<br>
**5.** Перегрузка операторов.<br>
**6.** Создание собственных перечислений (enum).<br>
**7.** Обработка исключений - сейчас при исключении транспайлер выдает технический крах.<br>
